## 📚 第2步：创建第1个工作流（文献检索式生成）

### 2.1 进入Bot编辑页面

1. 打开 https://www.coze.cn
2. 找到您的Bot（ID: 7616837019163476014）
3. 点击「编辑」进入编辑页面

### 2.2 创建工作流

1. 在左侧菜单找到「工作流」
2. 点击「创建工作流」
3. 填写信息：
   - 名称：`文献检索式生成`
   - 描述：`根据研究问题自动生成PubMed/EMBASE等数据库的专业检索式`
4. 点击「确定」

### 2.3 配置输入变量

在「开始」节点，添加以下变量：

#### 变量1：
- 名称：`research_question`
- 类型：`String`
- 必填：✓ 勾选
- 描述：`研究问题，例如：胚胎植入前遗传学检测对IVF结局的影响`

#### 变量2：
- 名称：`search_strategy`
- 类型：`String`
- 必填：不勾选
- 默认值：`precise`
- 描述：`检索策略：precise（精确）/sensitive（敏感）/limited（限定）`

### 2.4 添加第一个LLM节点

#### 添加节点
1. 点击「+ 添加节点」
2. 选择「LLM」
3. 节点名称：`解析研究框架`

#### 配置模型
- 模型：选择 `deepseek-chat`（如果没有，选择其他模型）
- Temperature：`0.3`

#### 复制System Prompt
1. 打开配置工具页面（http://localhost:5000/coze-config.html）
2. 点击「检索式生成」标签
3. 找到「节点1：解析研究框架」
4. 点击「📋 复制 System Prompt」按钮
5. 回到Coze页面，粘贴到System Prompt框

**System Prompt内容**：
```
你是一位循证医学专家，精通多种研究问题框架。

## 常见研究框架

### 1. PICO（干预性研究）
- P (Population): 研究人群
- I (Intervention): 干预措施
- C (Comparison): 对照措施
- O (Outcome): 结局指标

### 2. PECO（暴露研究/观察性研究）
- P (Population): 研究人群
- E (Exposure): 暴露因素
- C (Comparison): 对照/非暴露
- O (Outcome): 结局指标

### 3. SPIDER（定性/混合研究）
- S (Sample): 样本
- PI (Phenomenon of Interest): 感兴趣的现象
- D (Design): 研究设计
- E (Evaluation): 评估方法
- R (Research type): 研究类型

## 任务
1. 首先判断研究问题属于哪种类型
2. 提取相应的要素
3. 如果问题宽泛，**主动推断和扩展**可能的要素
4. 识别关键概念和同义词
5. 提供相关的MeSH主题词

【重要】即使问题不完整，也要尽可能推断和扩展，而不是返回空值。
```

#### 复制User Prompt
1. 在配置工具页面，点击「📋 复制 User Prompt」
2. 粘贴到Coze的User Prompt框

**User Prompt内容**：
```
研究问题：{{start.research_question}}

请分析研究类型，提取关键要素，并尽可能扩展相关概念。如果问题宽泛，请主动推断可能的PICO/PECO要素。
```

### 2.5 添加第二个LLM节点

#### 添加节点
1. 再次点击「+ 添加节点」
2. 选择「LLM」
3. 节点名称：`生成检索式`

#### 配置模型
- 模型：`deepseek-chat`
- Temperature：`0.3`

#### 复制System Prompt
从配置工具页面复制第二个节点的System Prompt

#### 复制User Prompt
```
研究问题：{{start.research_question}}

提取的要素：
{{parse_framework.elements}}

请生成精确的检索式，控制结果数量在合理范围内。
```

### 2.6 连接节点

1. 从「开始」节点拖拽连接线到「解析研究框架」节点
2. 从「解析研究框架」节点拖拽连接线到「生成检索式」节点
3. 从「生成检索式」节点拖拽连接线到「结束」节点

### 2.7 保存并测试

1. 点击右上角「保存」按钮
2. 点击「测试」按钮
3. 输入测试问题：
   ```
   PGT对IVF结局的影响
   ```
4. 点击「运行」
5. 查看输出结果

## 📚 第3步：创建其他4个工作流

### 工作流2：文献AI初筛
- 名称：`文献AI初筛`
- 描述：`使用AI根据研究问题对检索到的文献进行初步筛选`
- 输入变量：`research_question`（String，必填）

### 工作流3：文献数据提取
- 名称：`文献数据提取`
- 描述：`从文献内容中提取Meta分析所需的结构化数据`
- 输入变量：
  - `research_question`（String，必填）
  - `literature_content`（String，必填）
  - `outcome_of_interest`（String，选填）
- 节点：1个LLM节点
- Prompt：从配置工具复制

### 工作流4：文献质量评估
- 名称：`文献质量评估`
- 描述：`使用Cochrane RoB 2.0或Newcastle-Ottawa量表评估文献质量`
- 输入变量：
  - `literature_content`（String，必填）
  - `study_type`（String，选填，默认：rct）
- 节点：2个LLM节点（Cochrane RoB和Newcastle-Ottawa）
- Prompt：从配置工具复制

### 工作流5：Meta分析解读
- 名称：`Meta分析解读`
- 描述：`执行固定/随机效应模型Meta分析，计算合并效应量和异质性`
- 输入变量：多个（模型类型、效应量、置信区间等）
- 节点：1个LLM节点
- Prompt：从配置工具复制

### ❓ 常见问题

**Q: 找不到deepseek-chat模型怎么办？**
A: 可以选择其他模型，如GPT-4、Claude等，Prompt通用。

**Q: 变量名必须完全一致吗？**
A: 是的，变量名必须完全一致，否则无法正确传递数据。

**Q: 节点连接有顺序吗？**
A: 有顺序，必须按照逻辑顺序连接：开始 → 处理节点 → 结束。
